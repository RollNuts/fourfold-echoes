# D-020 Implementation Directive

Status: current D-020 directive.

この文書は、D-020 の現行仕様を実装へ渡すための公開可能な単一指示書である。
ローカル環境、個人情報、非公開キー、秘密情報は含めない。ここにない新規システムは追加しない。

## 仕様固定メモ

- FOURFOLD ECHOES は Steam 先行、買い切り、シングルプレイヤーのコンパクトなトップダウン・アクションアドベンチャーである。
- MVP は 1 ハブ、最大 3 リージョン、最大 4 ボス、探索ツール 1 種に固定する。
- 奥行きは、同じ探索ツールを部屋、敵、近道、報酬、ボスで繰り返し使わせる設計から作る。
- 実装は D-020 の基礎だけに限定する。移動、カメラ、通常攻撃、回避、被ダメージ、リトライ、探索ツール、探索ノード、部屋進行、近道、ローカルセーブ、最小 UI、BGM/SFX が対象である。
- オープンワールド、昼夜、釣り、農業、拠点建設、インベントリ、クラフト、クエストログ、ソーシャル、マルチプレイヤー、ライブサービスは入れない。
- マーケット検証用のスクリーンショット、動画、音声には、グレーボックスや仮音を完成物として使わない。
- 古い Echo Phase、抽出、広域探索、複数ツール前提の資料は D-020 を上書きしない。

## 実装順

1. この D-020 指示書を固定し、矛盾する古い方向性を実装判断に使わない。
2. `Bootstrap`、`PersistentSystems`、`Hub_Crossroads`、`Region_01_GreenRuins` のテスト部屋を用意する。
3. トップダウン移動、カメラ、通常攻撃、回避、被ダメージ、死亡、リトライ、敵 1 種を通す。
4. 1 種だけの `ExplorationTool` と `ExplorationNode` を実装し、道の解放か対象 reveal を画面上で読めるようにする。
5. `RoomController`、近道解放、ギミック部屋 2 つ、ローカルセーブ/ロードを進行フラグ単位で通す。
6. バーティカルスライスとして、ハブ 1、リージョン一部、通常敵 2 種、ミニボス 1、ボス 1、近道 1、ギミック部屋 2、レリック報酬 2、最小 UI、BGM 2、最低限の SFX を統合する。
7. スライスの見た目をスタイライズド 3D に置き換え、グレーボックスを検証素材から外す。
8. 仮音をマイルストーン品質の BGM/SFX に置き換え、音声なしでは読めない状態をなくす。
9. Steam Windows 先行の検証、Steam Deck 想定の画面/入力確認、スクリーンショット候補、短尺動画候補を実機ビルドから作る。

## スコープ外一覧

| 項目 | 理由 |
| --- | --- |
| オープンワールド、シームレス広域配信 | コンパクトな手作り部屋設計と矛盾する |
| Echo Phase や複数世界状態切替 | 1 ツール習熟から焦点が外れる |
| 昼夜サイクル | コンテンツ量と QA を増やす |
| 釣り、農業、拠点建設、サバイバル | 別ジャンルのループになる |
| インベントリ、クラフト、装備経済 | UI と経済設計が MVP を膨らませる |
| クエストログ | 許容量以上の物語/依頼構造を示唆する |
| ソーシャル、オンライン、アカウント、マッチメイク | シングルプレイヤー買い切りと合わない |
| マルチプレイヤー、協力、PvP | 設計と QA が倍増する |
| ライブサービス、日課、課金通貨、ガチャ | Steam 先行買い切りと合わない |
| 2 種目以降の探索ツール | 1 ツール習熟を薄める |
| 1 ハブ超過、3 リージョン超過、4 ボス超過 | D-020 の上限を超える |
| 手続き生成ワールド | 手作りの部屋読みを弱める |
| 複雑コンボを中核にする戦闘 | 読みやすい位置取り戦闘から外れる |

## 完成条件チェックリスト

- [ ] 30 分以内のプレイで、ハブ、リージョン部屋、近道、報酬、ボスまで到達できる。
- [ ] 静止画 1 枚で、プレイヤー、敵、探索ツール対象、進行方向が読める。
- [ ] 30 秒の実機動画で、移動、戦闘、ツール使用、部屋解決、報酬、ボス脅威が伝わる。
- [ ] 通常攻撃、回避、被ダメージ、敵予兆、ボス予兆に SFX または明確な視覚反応がある。
- [ ] 2 つのギミック部屋が同じ探索ツールを異なる配置で使う。
- [ ] 近道 1 つが開き、移動構造がコンパクトに折り返す。
- [ ] レリック報酬 2 つが、インベントリ画面なしで取得と効果を伝える。
- [ ] ローカルセーブ/ロードで、近道、ボス、報酬、リージョン解放フラグが復元される。
- [ ] 最小 UI は HP、ツール状態、プロンプト、ボス HP、ポーズ/設定だけに収まる。
- [ ] マーケット検証用素材にグレーボックス、仮音、未承認素材が含まれない。

## リスク表

| リスク | 重大度 | 兆候 | 対応 |
| --- | --- | --- | --- |
| 1 ツールが単調になる | 高 | 部屋ごとの差が配置だけになる | 部屋形状、敵配置、近道、ボス opening で使い方を変える |
| 戦闘の読みやすさが不足する | 高 | 被弾理由が分からない | 予兆時間、ヒット範囲、SFX、VFX を先に直す |
| スコープが膨らむ | 高 | 新システム提案が増える | 却下基準に照らし、既存項目の改善へ戻す |
| アートが市場検証に届かない | 高 | スクリーンショットがグレーボックスに見える | ハブ、主人公、敵、ボス、ツール対象の優先順位で置き換える |
| 音声が後回しになる | 中 | 無音または仮音の動画しか作れない | 操作、被弾、ツール、報酬、ボス cue を早期に入れる |
| Steam Deck 想定の UI が読みにくい | 中 | 1280x800 で文字や HUD が潰れる | HUD サイズ、コントローラー入力、設定を早期確認する |
| セーブの不具合で進行が壊れる | 高 | 近道や報酬が再現しない | `SaveData` を進行フラグに限定して回帰テストする |

## 市場検証可能性メモ

- 検証対象は「1 ツール、3 リージョン上限、4 ボス上限のコンパクトなトップダウン・アクションアドベンチャー」が Steam で伝わるかである。
- 最初の検証素材は実機プレイ由来に限定し、コンセプトアートや未実装説明だけで期待値を作らない。
- 30 秒動画では、ハブ、部屋問題、探索ツールの反応、敵の読み、報酬、ボス予兆を見せる。
- スクリーンショットは、主人公、ツール対象、敵、地形、進行方向が一目で分かるものを優先する。
- Steam ページ相当の文言は、実装済みまたは D-020 上限内で確定している内容だけを書く。
- 反応が弱い場合も、新システム追加ではなく、部屋密度、敵予兆、ボス見せ場、アート、音声、操作感を改善する。

## 却下基準

以下のどれかに当てはまる提案は却下する。

- インベントリ、クラフト、クエストログ、ソーシャル、オンライン、広域マップ、拠点建設を追加する。
- オープンワールド、複数世界状態切替、マルチプレイヤー、ライブサービスを別名で追加する。
- 2 種目の探索ツールを必要とする。
- 最初の 30 分のプレイ価値を上げない。
- 30 秒動画やスクリーンショットの明瞭さを上げない。
- QA コストがプレイ価値を上回る。
- アートや音声の完成を遅らせる。
- バーティカルスライスの完成を遅らせる。
- 現在の 1 ツールループで解くべき問題を新システムで逃がす。
- 「1 入れ 2 出し」を新システム追加の免罪符にする。これは D-020 内の内容や polish の交換にだけ使える。
