# Performance Budget

## Target

- PC baseline: 1080p / 60fps.
- Steam Deck compatibility is a target, not yet verified.
- PS5 port-readiness means sane asset budgets, not current platform support.

## Budgets To Establish

| Area | First Budget |
| --- | --- |
| frame time | 16.6ms target, record spikes |
| hero mesh | under 12k triangles until final hero pass justifies more |
| common enemy | under 8k triangles |
| first boss | under 60k triangles LOD0 with LODs |
| material slots | minimize; registry records budget |
| textures | 1k common, 2k hero/key props, 4k only for justified hero/boss |
| draw calls | measure after first production scene |
| loading | first demo area under acceptable player wait; measure later |

## Required Reports

- build size
- scene object count
- missing material/reference scan
- screenshot capture
- performance snapshot from a representative scene

Current scripted outputs:

- `artifacts/Reports/performance-snapshot.json`
- `artifacts/Reports/performance-snapshot.md`

These files are generated by `Scripts/Validation/write_market_reports.mjs` from
Unity validation metrics, Blender pilot manifests, screenshot metadata, and the
current D-020 evidence build artifact when present. They are evidence snapshots,
not a substitute for a future frame-time profiler capture.
