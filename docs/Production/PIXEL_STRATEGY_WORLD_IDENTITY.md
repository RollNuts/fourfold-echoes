# Pixel Strategy World Identity

Status: identity and anti-copy guardrail for the pixel strategy Steam screen.

## Core Premise

The screen is not a road-loop adventure. It is a Fourfold seal board where a
state-made "chosen one" myth has already gone bad: the crown hands a pawn a
wood stick and 12G, calls that destiny, and points him at an impossible threat.

The player does not become that hero, build a dungeon ecology, or run a generic
card expedition. The player operates the Fourfold Charter around the doomed
errand: fold routes, seal pressure, cut gates open, and decide whether to spare
the chosen debt or profit from it.

## Required First Read

- A tiny royal absurdity: wood stick, 12G, and an oversized demand.
- A Fourfold-specific object: four corner seals, a cross-fold, echo cells, and
  a cracked oath beat.
- A clear consequence: one decision changes the board now and makes the next
  beat more dangerous.
- A gate decision that feels like cutting a seal, not simply walking around a
  road loop.

## Visible Vocabulary

Use visible words that belong to this setting:

- `ECHO` for repeated pressure beats.
- `SEAL HAND` for placed seal options.
- `ROYAL KIT` for the insulting starter loadout.
- `CHOSEN DEBT` for the false prophecy burden.
- `OATH HEAT`, `OATH CRACK`, `GATE SEAL`, and `RELIC HEAT` for pressure events.

Avoid visible words that push the screen toward another game's read:

- `LOOP`, `TILE HAND`, `DECK`, `CAMP`, `EXPEDITION`, or `ROAD`.
- generic fantasy quest text that could be lifted into any classic RPG.
- monster-management language that implies dungeon ecology or room digging.

## Anti-Copy Boundary

- Do not copy a road-loop, camp-expedition, deck-table, or tile-card UI as the
  market-facing composition. The prototype can keep closed-path simulation
  internally, but the screenshot target must read as a folded seal diagram.
- Do not copy dungeon-management or monster-placement identity. Lairs and
  hazards are pressure marks on the charter, not a managed underground ecology.
- Classic RPG references are allowed only as readable folklore shorthand:
  absurd king, cheap starter gear, brave-but-doomed pawn. Do not copy protected
  monsters, classes, menu layouts, spell names, or mascot silhouettes.
- Loss-aversion tension is allowed as risk language, but visible UI should
  frame it as debt, oath, relic heat, and gate cuts.

## Screen Acceptance

A Steam capture should be rejected if a first-time viewer can honestly describe
it as "that road-loop card game with different sprites." It should pass only
when the first read is "a Fourfold seal board about exploiting or sparing an
unfairly chosen pawn."
